## 추상메서드

추상메서드는 선언만 있고 구현이 없는 메서드이다. abstract 키워드로 선언하며 본문 대신 세미콜론으로 끝난다.

```java
public abstract void printNamt();
```

---

## 추상클래스

두 가지 경우로 나뉜다:
1. 추상 메서드를 포함하는 경우
2. 추상 메서드가 없는 경우

추상클래스는 객체를 생성할 수 없다.

```java
abstract class Person{
    public abstract void walk();
    public abstract void run();
    public void Person() {}
}

class Student extends Person {
    public void walk() {
        System.out.println("Walking");
    }
    public void run() {
        System.out.println("Running");
    }
}
```

---

## 인터페이스(interface)

추상클래스보다 더 간결한 표현 방식이다.

```java
interface Person {
    public static final int life = 100;
    abstract public int getAge();
    abstract public String getName();
}

class Student implements Person, Animal {
    // 구현 코드
}
```

### 주요 특징
- implements 키워드로 구현
- **다중 구현 가능** (클래스와 달리 여러 인터페이스 동시 구현 가능)
- 모든 메서드는 추상메서드
- 상수만 선언 가능
