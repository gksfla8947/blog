# 측정 프레임워크(MF, Measurement Framework)

Y축 측정 프레임워크는 여러 요소를 통해 프로세스 달성 척도를 평가함

- 프로세스 능력 수준

![프로세스 능력 수준 2](/images/notion/aspice/image-4.png)

- 프로세스 속성

![프로세스 속성](/images/notion/aspice/image-5.png)

	- 9개의 프로세스 속성

- 프로세스 속성 등급 : **프로세스 속성**의 등급 달성을 측정
	- 척도
		- N(Not) : 미달성 - (0% to 15%)
		- P(Partially) : 부분적 달성 - (15% to 50%)
			- P- (15% to 32.5%)
			- P+ (32.5% to 50%)
		- L(Largely) : 대부분 달성 - (50% to 85%)
			- L- (50% to 67.5%)
			- L+ (67.5% to 85%)
		- F(Fully) : 완전히 달성 (85% to 100%)
	- 등급 산출 방법
		- R1
			- 평가 범위 내 **각 프로세스의 성과**는 검증된 자료를 기반으로 **프로세스 인스턴스**에 대해 특성화 되어야 함
			- 평가 범위 내의** 각 프로세스에 대한 각 프로세스 속성의 성과**는 검증된 자료를 기반으로 **프로세스 인스턴스**에 대해 특성화 되어야 함
			- 평가된 모든 **프로세스 인스턴스**에 대한 **프로세스 성과 특성**은 프로세스 **수행 속성**에 대한 달성 등급을 제공하기 위해 집계되어야 한다
			- 평가된 모든 **프로세스 인스턴스**에 대한 **프로세스 속성 성과 특성**은 **프로세스 속성 달성 등급**을 제공하기 위해 집계되어야 한다
		- R2
			- 평가 범위 내의** 각 프로세스에 대한 각 프로세스 속성의 성과**는 검증된 자료를 기반으로 프로세스 인스턴스에 대해 특성화 되어야 함
			- 평가된 모든 프로세스 인스턴스에 대한 **프로세스 속성 성과 특성**은 **프로세스 속성 달성 등급**을 제공하기 위해 집계되어야 한다
		- R3
			- 평가된 프로세스 인스턴스에 대한 프로세스 속성 등급 산출은 **집계 없이 수행**되어야 한다
	- 집계 방법
		- 1차원 수직형 집계
			- 대상 프로세스의 프로세스 속성에 대해 등급 산출 시, 심사자는 관련된 프로세스 속성 성과의 등급을 집계한다(수직적 집계로써 수행됨)
		- 1차원 수평적 집계
			- **다양한 프로세스 인스턴스를 통해** 대상 프로세스의 프로세스 속성 성과에 대해 등급 산출 시, 심사자는 대상 프로세스 속성 성과에 대한 관련된 프로세스 인스턴스의 등급을 집계한다(수평적 집계로써 수행됨)
		- 2차원 행렬형 집계
			- 대상 프로세스의 프로세스 속성에 대해 등급 산출 시, 심사자는 모든 프로세스 인스턴스에 대한 모든 프로세스 속성 성과의 등급을 집계한다. 그러한 집계는 등급 산출의 모든 범위에 대한 행렬 집계로써 수행된다

# 프로세스 평가 모델 (PAM, Process Asessment Model)

**프로세스 성과와 프로세스 속성 성과가 프로젝트와 조직 단위의 인스턴스화된 프로세스에 존재하는지 식별하기 위해 지표를 제공**

- 능력 수준의 판단을 지원하는 데 필요한 객관적인 증거를 수집하는데 있어 심사자에게 도움
- 이러한 지표는 의무적으로 반드시 준수되어야 하는 점검표로 고려하려는 것은 아님

프로세스 성과와 달성의 존재 여부를 판단하기 위해 평가는 객관적인 증거를 얻는다

- 모든 객관적인 증거
	- 평가된 프로세스의 작업 산출물과 저장소 내용에 대한 조사로부터 도출
	- 평가된 프로세스의 수행자와 관리자의 증언으로부터 도출

프로세스 평가 모델 지표의 두 가지 유형

- 프로세스 수행 지표
	- 능력 수준 1에만 적용
	- 프로세스 성과의 **이행 정도**에 대한 표시 제공
- 프로세스 능력 지표
	- 능력 수준 2에서 5까지 적용
	- 프로세스 속성 달성의 이행 정도에 대한 표시 제공

평가 지표는 평가 동안 수집된 증거에 따라 어떤 사례가 수행되었는지 확인하기 위해 사용된다
객관적인 증거가 프로세스 수행에 대한 증거를 제공하듯이, 프로세스 능력 지표의 존재는 프로세스 능력의 증거를 제공한다

## 프로세스 수행 지표

기본 사례 (BP)
작업 산출물 (WP)

- 둘 다 하나 이상의 프로세스 성과와 관련이 있다
	- BP와 WP는 항상 프로세스에 특화되며 일반화되지 않음
	- BP는 활동 지향적인 지표를 나타내고
	- WP는 결과 지향적 지표를 나타냄
- BP와 WP는 평가 수행 시 수집하고 축적하는 객관적인 증거를 판단하는데 사용
⇒ 심사자가 사용할 수 있는 대안적인 지표이다

프로세스 평가 모델은 각각의 WP에 대한 작업 산출물 특성(WPC)를 제공함

- WP와 WPC는 평가 동안 신속하게 접근할 수 있는 정보
	- 그런 측면에서 WP와 WPC는 단지 예시적인 구조를 나타냄
	- 엄격한 필수 사항도 아니며 조직을 위한 준수사항도 아님
	- 대신 이행된 프로세스에 대한 구체적인 작업 산출물과 문서의 실제 구조, 형식, 내용은 프로젝트와 조직에서 각각 정의해야 함
	- **프로젝트 및 조직은 WP가 의도된 목적과 요구사항에 적절하고, 개발 목표와 관련하여 적절한지 보장하여야 한다.**

## 프로세스 능력 지표

일반 사례 (GP)
일반 자원 (GR)

- 둘 다 하나 이상의 PA 달성에 관련된다
	- 프로세스 능력 지표는 일반적인 형태이다 ⇒ 어떤 프로세스에도 적용됨
- GP : 객관적인 증거를 판단하기 위해 **활동 지향적인 지표**를 나타냄
- GR : 객관적인 증거를 판단하기 위해 **인프라 지향적인 지표**를 나타냄
- 심사자는 평가 과정에서 프로세스 능력 지표를 뒷받침하는 증거를 수집하고 축적해야 함
⇒ 심사자가 사용할 수 있는 대안적인 지표이다

프로세스의 능력 수준 1은 프로세스 성과가 달성되는 정도의 측정에 의해서만 특정되지만
측정 프레임워크는 각 수준이 프로세스 속성을 드러내 보일 것을 요구하며 그에 따라 프로세스 평가 모델이 적어도 하나의 프로세스 능력 지표를 도입할 것을 요구한다
